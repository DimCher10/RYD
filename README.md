# RYD

RYD — полнофункциональное веб-приложение для планирования подготовки к олимпиадам по математике и информатике.

## Возможности

- регистрация и вход с изолированными данными каждого пользователя;
- олимпиады с датой и личной целью;
- темы по математике, информатике и искусственному интеллекту;
- собственные задачи, сложность, срок и связь с темой/олимпиадой;
- текстовые конспекты для каждой темы;
- плановое и фактическое время для каждой задачи, сумма времени по темам;
- дневная и 30-дневная статистика времени с процентами по темам;
- листки по математике и контесты по информатике/ИИ с учетом общего числа задач и решенных;
- статистика решенных задач за день, месяц и все время, включая круговую диаграмму по темам;
- условие задачи в виде текста или фотографии;
- статусы «в плане», «в работе», «решено»;
- обзор текущей подготовки и статистика за 30 дней;
- адаптивный интерфейс для компьютера и телефона.
- отдельная вкладка ежедневного плана до 1 июня 2027 года с сохранением отметок выполнения;
- три тематических тренажера на каждый день: практика, устный ответ и контроль качества;
- подробное расписание дня с длительностью блоков, модулем курса и прямыми ссылками на материалы;
- космическая AI-тема интерфейса с адаптивной навигационной консолью;
- подборка ссылок для подготовки к ML, SQL, алгоритмическим и NLP-собеседованиям.

## Локальный запуск

Требуется Python 3.10 или новее. Внешние зависимости не нужны.

```powershell
py server.py
```

Откройте `http://127.0.0.1:8000`. При регистрации создаются несколько стартовых тем и задач, которые можно удалить.

## Docker

```powershell
docker compose up --build
```

SQLite хранится в Docker volume `ryd-data` и не теряется при пересоздании контейнера.

## Конфигурация

| Переменная | По умолчанию | Назначение |
| --- | --- | --- |
| `HOST` | `127.0.0.1` | Интерфейс прослушивания |
| `PORT` | `8000` | HTTP-порт |
| `RYD_DB_PATH` | `data/ryd.db` | Путь к SQLite |
| `RYD_SECURE_COOKIE` | `0` | Установите `1` при работе через HTTPS |

Для публикации на сервере запускайте контейнер за HTTPS reverse proxy (Caddy, nginx или облачная платформа), подключив постоянный volume к `/app/data`.
